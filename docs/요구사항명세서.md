## 1. 서비스 개요

### 1.1 한 줄 정의
한국 조별리그가 끝난 상태에서, **다른 조 경기 결과에 따라 한국의 32강 진출 가능성이 어떻게 변하는지** 실시간으로 추적하고, 전체 대진표와 오늘의 경기 일정을 한눈에 확인하는 원페이지 앱.

### 1.2 앱 이름
**레드볼** (영문: RedBall)

### 1.3 핵심 목적
- "다른 조에서 이런 결과가 나오면 우리가 올라간다"를 한눈에 파악
- 12개 조의 3위 팀 비교를 자동 계산해서 실시간 갱신
- 경기가 진행될 때마다 진출 확률이 변하는 걸 인터랙티브하게 체감

### 1.4 해결하는 문제
- 한국은 조별리그 종료 (A조 3위 가능성) → 상위 8개 3위 팀에 들어야 32강
- 48팀/12조 신포맷이라 다른 조 결과까지 봐야 해서 경우의 수가 매우 복잡
- 뉴스마다 경우의 수 설명이 다르고 업데이트가 느림
- 경기 실시간 중 "지금 이 경기에서 골이 들어가면 우리 진출이야?"를 즉시 알고 싶음

### 1.5 타깃 사용자
**월드컵 기간 중 한국 대표팀의 32강 진출을 응원하는 축구 팬** — 연령/성별 무관

## 2. 2026 월드컵 포맷 정리

### 2.1 조별리그
- **48팀 / 12조 / 조당 4팀**
- 각 조 3경기 (라운드 로빈)
- **한국: A조** (대한민국, 체코, 멕시코, 남아프리카공화국)

### 2.2 한국 A조 경기 결과 (조별리그 종료)
| 날짜 | 경기 | 결과 |
|------|------|------|
| 6/12 (금) 11:00 | 대한민국 vs 체코 | **2-1 승** |
| 6/19 (금) 10:00 | 멕시코 vs 대한민국 | **1-0 패** |
| 6/25 (목) 10:00 | 남아공 vs 대한민국 | **1-0 패** |

**한국 성적**: 1승 2패 · 승점 3 · 득 2 · 실 3 · 골득실 -1

### 2.3 진출 조건
- **조 1~2위**: 자동 32강 진출 (24팀)
- **조 3위**: 12개 조 중 상위 8팀 → 32강 진출 (8팀)
- **조 4위**: 탈락
- **한국 상황**: A조 3위 가능성 → 다른 조 3위와 비교해서 상위 8팀에 들어야 32강

### 2.4 32강 이후
- 32강 → 16강 → 8강 → 4강 → 결승

### 2.5 순위 결정 기준 (FIFA 규정)
1. 승점 (승3/무1/패0)
2. 골득실차
3. 다득점
4. 직접 대결 결과
5. 페어플레이 포인트
6. FIFA 랭킹

## 3. 핵심 기능 A — 진출 가능성 대시보드 (상단)

### 3.1 현재 상태 배너
- **대형 상태 표시**: "32강 진출 확정" / "32강 진출 가능 (XX%)" / "탈락 확정"
- 한국 성적 요약: 1승 2패 · 승점 3 · 골득실 -1
- **실시간 진출 확률 (%)**: 다른 조 남은 경기 결과 조합 기반 계산
- 경기가 진행될 때마다 확률이 실시간 변동

### 3.2 12개 조 3위 비교표
- 한국(A조 3위)과 다른 11개 조의 3위 팀 실시간 비교
- 승점 / 골득실 / 다득점 기준 정렬
- **상위 8팀 컷라인** 표시 (8위와 9위 사이 빨간 줄)
- 한국 행 하이라이트 + 현재 몇 번째인지 표시
- 아직 조별리그 진행 중인 조는 "진행 중" 표시

### 3.3 경우의 수 시나리오 카드
- **다른 조의 남은 경기** 결과에 따라 한국 진출 여부가 바뀌는 시나리오
- 예: "F조 3위가 승점 1로 끝나면 → 한국 32강 진출"
- 예: "G조 3경기에서 무승부가 나오면 → 한국에게 유리"
- 각 시나리오에 **"진출" / "탈락"** 결과 표시
- 현재 진행 중인 경기의 스코어에 따라 카드가 실시간 변동

### 3.4 주요 관전 포인트
- "오늘 이 경기를 주목하세요" — 한국 진출에 영향을 주는 경기 하이라이트
- 해당 경기의 어떤 결과가 한국에게 유리한지 표시

## 4. 핵심 기능 B — 대진표 (중단)

### 4.1 조별리그 대진표
- 12조 전체를 한눈에 볼 수 있는 뷰
- **E조(한국) 하이라이트** — 다른 조보다 강조 표시
- 각 조의 현재 순위표 간략 표시

### 4.2 토너먼트 대진표
- 32강 → 16강 → 8강 → 4강 → 결승 트리 구조
- 한국의 예상 경로 하이라이트
- 진행된 경기는 결과 표시, 미진행 경기는 대진 표시

---

## 5. 핵심 기능 C — 오늘의 경기 (하단)

### 5.1 오늘 경기 일정
- 오늘 날짜 기준 경기 리스트
- 경기 시간 (한국 시간 KST)
- 팀 국기 + 팀명
- **한국 경기는 최상단에 강조 표시**

### 5.2 경기 결과
- 진행 중: "전반 35분 · 1-0" 같은 실시간 스코어
- 종료: 최종 스코어
- 예정: 킥오프 시간

### 5.3 날짜 이동
- 좌우 화살표로 다른 날짜 경기 확인

## 6. 데이터 소스

- **Football-Data.org** (무료 티어: 10 req/min)

### 6.2 폴백

### 6.3 업데이트 주기

- 경기 없는 시간: 30분 간격
- 유저가 당겨서 새로고침 가능

## 7. 경우의 수 계산 로직

### 7.1 핵심 계산
- 한국은 A조 3위 확정 (승점 3, 골득실 -1, 득점 2)
- **다른 11개 조**의 남은 경기 결과 조합(승/무/패) 생성
- 각 조합에서 3위 팀의 승점/골득실/득점 계산
- 12개 조의 3위 팀을 정렬 → 한국이 상위 8팀에 드는 조합 필터링
- **진출 확률 = 한국이 상위 8위 안에 드는 조합 수 / 전체 조합 수**

### 7.2 실시간 업데이트
- 경기 결과가 확정될 때마다 남은 조합 수가 줄어들며 확률 재계산
- 진행 중인 경기의 현재 스코어도 반영 (가정: 현재 스코어가 최종 결과일 때)
- "이 경기에서 골이 들어가면 확률이 X%→Y%로 변합니다" 표시

### 7.3 시나리오 요약
- 전체 조합 대신 **핵심 조건**을 자연어로 요약
- "B조 3위 승점이 3 이하면 한국 진출" 같은 직관적 문장
- 가장 유리한 시나리오 / 가장 현실적 시나리오 / 최악의 시나리오 분류

### 7.4 성능 고려
- 조합 수가 매우 클 수 있으므로 서버에서 사전 계산 후 캐싱
- 경기 결과 확정 시마다 재계산 (폴링 시 매번 계산 X)

## 10. 기술 스택

### 10.1 프런트엔드
- **프레임워크**: `@apps-in-toss/web-framework`
- **베이스**: React 18 · Vite · TypeScript
- **차트/시각화**: 직접 구현 (CSS Grid + Flexbox)

### 10.2 백엔드
- **런타임**: Node.js + Express + TypeScript
- **DB**: JSON 파일 기반 (캐싱용)
- **호스팅**: Railway → NHN Cloud VM (7월 이후)
- **인증**: 토스 로그인 (최소한)

### 10.3 외부 서비스
- **축구 API**: Football-Data.org (무료) 또는 대안
- **국기 아이콘**: flagcdn.com (무료)
